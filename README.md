# Каталог тренеров — Академия Softline

Внутренний каталог компетенций тренеров по направлению **ИИ / ML / Big Data**. Используется сейлами и маркетингом для подбора спикеров под бриф клиента и подготовки материалов (КП, лендинги, презентации).

В каталоге **15 тренеров** с поиском, фильтрами по целевой аудитории и темам, готовыми к копированию краткими и подробными биографиями.

## Как открыть

Двойным кликом по `index.html` — страница откроется в браузере. HTML самодостаточный: всё (данные, фотографии, стили, скрипты) встроено в один файл, ничего скачивать дополнительно не нужно.

Если репозиторий опубликован через **GitHub Pages**, каталог будет доступен по ссылке (см. раздел «Публикация» ниже).

## Структура папки

```
.
├── index.html                  ← главная страница каталога (открывается в браузере)
├── README.md                   ← этот файл
├── .gitignore
├── source/                     ← исходные данные для обновления
│   ├── Каталог_тренеров.xlsx   ← Excel-источник: правьте здесь
│   └── trainers.json           ← структурированные данные (автоматически)
└── photos/                     ← фотографии тренеров в полном разрешении
    ├── pisarenko.jpg
    ├── zhukov.jpg
    └── ...
```

## Как обновить каталог

**Сценарий 1 — поправить текст у тренера или добавить нового.**
Откройте `source/Каталог_тренеров.xlsx`, внесите правки в нужную строку (или добавьте новую), сохраните. Затем нужна пересборка `index.html` из обновлённого Excel — пришлите файл, помогу пересобрать.

**Сценарий 2 — заменить фотографию.**
Положите новый файл в `photos/` с тем же именем (например, `salovsky.jpg`). Затем — пересборка `index.html`.

**Сценарий 3 — мелкая правка прямо в HTML.**
HTML самодостаточный, и все тексты лежат внутри файла в JavaScript-объекте `TRAINERS`. В крайнем случае можно поправить точечно прямо там — но при следующей пересборке из Excel изменения затрутся, поэтому держите Excel в актуальном состоянии.

## Публикация через GitHub Pages (опционально)

Если хотите дать сейлам/маркетингу постоянную ссылку на каталог:

1. Зайдите в репозиторий → **Settings** → **Pages**.
2. В разделе «Build and deployment»: Source = **Deploy from a branch**, Branch = `main` (или `master`), папка `/ (root)`.
3. Сохраните. Через 1–2 минуты страница будет доступна по адресу вида `https://<организация>.github.io/<репозиторий>/`.

**Важно про приватность:** GitHub Pages по умолчанию делает страницу **публичной** — доступной любому в интернете, у кого есть ссылка. Если каталог должен быть только для сотрудников Softline, есть три варианта:

- Сделать репозиторий публичным и считать каталог открытым (биографии тренеров и так публичные — на их LinkedIn/сайтах).
- Использовать **приватный репозиторий + GitHub Pages для приватных репо** (доступно на платных тарифах GitHub Team / Enterprise — авторизация по аккаунту GitHub членов организации).
- Не использовать Pages: репозиторий приватный, члены команды клонируют папку и открывают `index.html` локально.

## Что НЕ загружать в этот репозиторий

В репозитории сознательно нет оригинальных резюме тренеров (`.docx`, `.pdf`, `.doc`). В них — личные контакты, телефоны, история работы за пределами Академии. Эти файлы должны храниться отдельно, в защищённом хранилище (Я.Диск/Google Drive/корпоративный SharePoint с правами доступа), а не в git.
